# Оверлей Android-приложения (тонкий клиент)

Мобильное приложение SiYuan живёт в отдельном репозитории
[`siyuan-note/siyuan-android`](https://github.com/siyuan-note/siyuan-android).
Чтобы не форкать его целиком ради небольшой правки, мы храним изменённые файлы
здесь и **накладываем их поверх** апстрима во время сборки APK в CI
(`.github/workflows/master-build.yml`, джоб `Android APK`).

## Что добавляет оверлей

Режим **тонкого клиента**: на первом запуске приложение спрашивает —
работать **локально** (ядро в телефоне, как в апстриме) или **подключиться к
серверу**. В режиме сервера приложение не поднимает локальное ядро, а грузит
веб-UI со шлюза (`<адрес>/`), где шлюз показывает свою форму входа
(`/gw/login`); после входа все данные — на сервере.

Изменён один файл: `app/src/main/java/org/b3log/siyuan/MainActivity.java`.

## Пин версии апстрима

CI берёт `siyuan-note/siyuan-android` по **фиксированному коммиту**
`9123095b83bb7da25e53dd4db42fdbf9718fc982` (тег `v3.7.2-alpha.2`), под который
сделан оверлей. Полная замена файла детерминирована только при совпадении базы.

### Как обновить под новую версию апстрима
1. `git clone` нужный тег `siyuan-note/siyuan-android`.
2. Перенести правки тонкого клиента в новый `MainActivity.java` (искать
   `remoteMode` / `showModeChooser` / `PREFS`).
3. Обновить файл здесь и `ref:` (SHA) в `master-build.yml`.
